## 什么是作用域

作用域是一套规则，用来确定在何处以及如何查找变量（标识符）

### 编译原理

#### 传统编译步骤

- 分词/词法分析

分词和词法分析的主要差异在于词法单元的识别是通过有状态还是无状态的方式进行

- 解析/语法分析

将词法单元流转换成抽象语法树（Abstract Syntax Tree, AST)，以`var a = 2`为例，转换成的AST结构大致如下：
```js
// 顶级节点
VariableDeclaration {
    Identifier: a,
    AssignmentExpression: {
        Numericliteral: 2
    }
}
```
- 代码生成

#### JavaScript编译步骤

JavaScript 引擎的编译要比传统的编译步骤复杂得多，因为 JavaScript 的编译过程不是发生在构建之前的，通常是编译后直接运行（间隔几微秒），所以引擎通常不会有大量的时间进行优化，从而需要通过 JIT、延迟编译、重编译等方法来提高性能

### 理解作用域

#### 基本概念

引擎：从头到尾负责整个 JavaScript 程序的编译及执行过程

编译器：引擎的好朋友之一，负责语法分析及代码生成等脏活累活

作用域：引擎的另一位好朋友，负责收集并维护由所有声明的标识符（变量）组成的一系列查询，并实施一套非常严格的规则，以确定当前执行代码对这些标识符的访问权限

#### LHS & RHS

LHS：查找赋值操作的目标（存值）

RHS：查找赋值操作的源头（取值）

#### 作用域嵌套

知道**原型链**吗？作用域嵌套类似原型链查找属性值的流程，若引擎在当前作用域中无法找到某个变量，就会在外层嵌套的作用域中继续查找，直至全局作用域（类同于原型链的 null

#### 异常

`ReferenceError`：作用域判别失败

`TypeError`：作用域判别成功，但对结果的操作非法或不合理
